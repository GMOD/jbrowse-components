# @jbrowse/shader-tools

The Slang → WGSL/GLSL + typed-TS shader **codegen** (`pnpm gen:shaders`). This
is a **dev/build tool**, not runtime code — nothing in the app imports it; it is
invoked by the root `gen:shaders` script and CI's
`gen:shaders && git diff --exit-code` staleness check.

## Files

- `src/build-shaders.ts` — the driver. Walks the whole repo for `*.slang`,
  fetches/caches the `slangc` binary, compiles each shader to WGSL (WebGPU) +
  GLSL ES 3.00 (WebGL2) + a reflection layout, and writes `<name>.generated.ts`
  next to each `.slang`. Also the package `bin` (`jbrowse-build-shaders`).
- `src/shader-codegen/codegen.ts` — pure transforms: reflection JSON + shader
  strings → the typed `.generated.ts` module (stride, field offsets, typed
  packer, `GL_ATTRIBUTES`).
- `src/shader-codegen/vulkanGlslToWebgl2.ts` — Vulkan-GLSL → WebGL2-GLSL fixups.

## Invariants

- **Repo-coupled, for now.** `build-shaders.ts` derives `REPO_ROOT` from its own
  location (three levels up from `src/`) and walks the whole workspace;
  `SHARED_INCLUDE` is hardcoded to `packages/core/src/gpu/shaders` (the shared
  `.slang` modules `hpmath`/`colorPack`). It is **not yet** parameterized for
  use in an external plugin repo — making it a portable third-party CLI (per
  ADR-030) means lifting those two assumptions into args/config first.
- **Never hand-edit `*.generated.ts`.** Edit the `.slang` source and run
  `pnpm gen:shaders`; CI's `git diff --exit-code` catches stale output. See
  ADR-005.
- **Keep the emit templates stable.** The provenance comment strings the codegen
  writes into every `.generated.ts` (e.g.
  `// AUTO-GENERATED by packages/shader-tools/src/shader-codegen from X.slang`)
  are part of the generated output — changing them regenerates ~40 files. Change
  deliberately.
- **The emitted HAL import is `@jbrowse/core/gpu/hal`** (a re-export shim →
  `@jbrowse/render-core/hal`). Repointing it at `@jbrowse/render-core/hal`
  directly would churn every `.generated.ts` and require every plugin to depend
  on render-core — deferred to the shim-retirement follow-up.

See ADR-005 (codegen design) and ADR-030 (why this is its own package + the
static-import-only policy).
