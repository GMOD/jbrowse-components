# @jbrowse/shader-tools

The Slang → WGSL/GLSL + typed-TS shader **codegen** (`pnpm gen:shaders`). This
is a **dev/build tool**, not runtime code — nothing in the app imports it; it is
invoked by the root `gen:shaders` script and by the lint job's "Check generated
shaders are up to date" step (`gen:shaders` then `git diff` on
`*.generated.ts`).

## Files

- `src/build-shaders.ts` — the driver. Walks the whole repo for `*.slang`,
  fetches/caches the `slangc` binary, compiles each shader to WGSL (WebGPU) +
  GLSL ES 3.00 (WebGL2) + a reflection layout, and writes `<name>.generated.ts`
  next to each `.slang`. Also the package `bin` (`jbrowse-build-shaders`).
- `src/shader-codegen/codegen.ts` — pure transforms: reflection JSON + shader
  strings → the typed `.generated.ts` module (stride, field offsets, typed
  packer, `GL_ATTRIBUTES`). Unit-tested in `codegen.test.ts`.
- `src/shader-codegen/vulkanGlslToWebgl2.ts` — Vulkan-GLSL → WebGL2-GLSL fixups.
  Unit-tested in `vulkanGlslToWebgl2.test.ts`.

## Validators

`naga` (WGSL) and `glslangValidator` (GLSL-ES) are **optional**: they don't
affect the generated output (that's driven entirely by the pinned `slangc` + the
codegen), only whether a malformed shader is caught at build time. Unlike
`slangc` they aren't auto-fetched, so the driver probes for each and skips it
with a one-line warning if absent — a contributor can always regenerate without
them. CI installs `glslang-tools` so GLSL validation runs there; set
`NAGA=''`/`GLSLANG=''` to force-skip locally.

## Invariants

- **Portable (published, per ADR-030).** The scan root is `--root=` / cwd, not
  this file's location, and `SHARED_INCLUDE` resolves
  `packages/render-core/src/shaders` first, then
  `node_modules/@jbrowse/render-core/src/shaders` (render-core ships
  `src/shaders` for this), with `--shared-include=` to override. In this repo
  `pnpm gen:shaders` runs from the root, so cwd is the repo root and behavior is
  unchanged. Don't reintroduce a path derived from `import.meta.url` — it
  resolves into the consumer's `node_modules` once installed.
- **Never hand-edit `*.generated.ts`.** Edit the `.slang` source and run
  `pnpm gen:shaders`; CI's `git diff --exit-code` catches stale output. See
  ADR-005.
- **Keep the emit templates stable.** The provenance comment strings the codegen
  writes into every `.generated.ts` (e.g.
  `// AUTO-GENERATED by packages/shader-tools/src/shader-codegen from X.slang`)
  are part of the generated output — changing them regenerates ~40 files. Change
  deliberately.
- **The emitted HAL import is `@jbrowse/render-core/hal`** (the leaf rendering
  package — `codegen.ts` writes this string into every `.generated.ts`).
  Consequently every package that owns a `.slang` must depend on
  `@jbrowse/render-core`; changing this import string regenerates ~40 files, so
  change it deliberately (this is the type-only HAL import, not a runtime dep of
  the codegen itself).

See ADR-005 (codegen design) and ADR-030 (why this is its own package + the
static-import-only policy).
