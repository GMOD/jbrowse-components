## Core design (the durable knowledge)

**Everything is computed from the MAF we already fetch ‚Äî no precomputed files.**
`plugins/maf/src/LinearMafGetAlignmentDataRpc/computeMafCoverage.ts` walks
block‚Üírow‚Üícolumn comparing each species base to the reference. In one pass it
now produces three things:

- `depths` ‚Äî coverage (unchanged; counts ALL rows).
- `identity: Float32Array` ‚Äî **aggregate** per-bp percent identity (one number
  per ref position, summed across species). Feeds the conservation band.
- `matchesPerRow` / `classifiablePerRow: Float32Array` (indexed by
  `rowIndex`, sized `nRows`) ‚Äî **per-row** counts. Feeds per-row identity.

**Identity policy (settled ‚Äî keep consistent):**
- Reference row is **excluded** (`refRowIndex`, resolved in the RPC from
  `region.assemblyName`). Otherwise a lone divergent species reads 50%, and the
  reference row reads a trivial 100%.
- Sample gaps excluded from the denominator; ref-`N` columns unclassifiable
  (‚Üí `NaN`); sample `N`/IUPAC counts as a mismatch (matches the SNP coloring).
- Comparison is uppercased (`& ~LOWER_BIT`) so soft-masking never affects it.

**Critical hazard ‚Äî refetch loop.** Per-row identity is derived from fetch
results (`rpcDataMap`). It MUST NOT flow into `sources` / `rpcProps`
(`orderedSampleIds`), or every data arrival changes the fetch args ‚Üí infinite
refetch. It's kept in a separate `rowIdentities` getter (sums per-row counts
over visible regions) consumed only by labels + tooltip. Do not "simplify" by
moving it onto `sources`.

**Data-flow contract.** Per-row identity ships as raw **counts** (not ratios)
so the main thread sums across visible regions before dividing. The aggregate
band ships the per-bp ratio and the draw fn does per-pixel mean
(`drawConservation.ts:accumulateConservation`, unit-tested).

**Kent cross-check result (important):** the UCSC `bigMafSummary` `score` we
render as zoom-out alpha bars is a normalized HOXD70 *alignment* score, NOT
percent identity (`hgLoadMafSummary.c:scorePairwise`). So our identity is
computed by us; the summary-bar comments are labeled accordingly. UCSC's real
per-species %id is a separately precomputed wiggle. See
`~/src/vendor/kent` (local checkout) if you need to re-verify.

## Display surfaces (where each thing lives)

- **Conservation band** ‚Äî `drawConservation.ts` + `MafConservationCanvas.tsx` +
  `MafConservationYScale.tsx`, stacked between coverage band and rows. Fill is
  `theme.palette.coverage` (grey700). Toggle: "Conservation (% identity)".
- **Per-row identity** ‚Äî `NN%` next to each row label via
  `packages/tree-sidebar/src/SvgRowLabels.tsx` (gained an optional per-row
  `details?: (string|undefined)[]`); MAF passes `model.rowIdentityLabels`. Also
  in the per-base hover tooltip (`MafHover.rowIdentity` ‚Üí
  `MafAlignmentTooltipContents`). Toggle: "Per-row identity %".
- **Layout invariant:** all rows-offset reads route through the single
  `rowsTopOffset` getter (= coverage + conservation band heights). Any new band
  must extend this, not add another literal offset.
- Both toggles **default off** (no snapshot/screenshot churn) and are stripped
  from snapshots by `stripDefault`.

## Screenshots / docs / test data

- Specs in `website/scripts/screenshot-specs.ts`: `maf_track` (intro, per-row
  identity on) and `maf_conservation` (two-stage menu‚Üíband). Both use a thin
  **local** config `CE_MAF = 'test_data/ce_maf.json'`.
- `test_data/ce_maf.json` wires the ce11 assembly (UCSC twobit, CORS-ok) + a
  **local tabix slice** of the real 26-way at a conserved exon
  (`ce11.26way.chrI_subset.bed.gz` + `.tbi` + `ce11.26way.nh`, extracted
  `chrI:2,996,000-3,004,000` ‚Äî has up to 26 species aligning). Local = fast +
  deterministic. To re-slice: `tabix https://jbrowse.org/demos/ce/ce11.26way.bed.gz <region>` ‚Üí bgzip ‚Üí `tabix -p bed`.
- **Regenerating screenshots needs a jbrowse-web rebuild first** (the generator
  renders the built bundle): `cd products/jbrowse-web && pnpm build`, then
  `cd website && node --experimental-strip-types scripts/generate-screenshots.ts --filter maf --force`.
  PNGs are ~2000px wide ‚Äî `convert x.png -resize 1400x /tmp/y.png` before Read.
  `build/test_data` is a real copy (NOT a symlink) ‚Äî copy new test_data files
  into it for an existing build, or rebuild.
- Lingering hover "Loading" in a capture = a tooltip at the default cursor;
  park it with `actions:[{type:'hover',from:{x:250,y:100}},{type:'delay',ms:2000}]`.
