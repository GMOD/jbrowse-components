## What this branch is

`webgl-poc` migrates JBrowse 2 from block-based HTML-canvas rendering to a GPU
pipeline (**WebGPU → WebGL2 → Canvas2D fallback**; SVG export via Canvas2D),
keeping every existing track type working on every backend. It is a foundational
rewrite of the rendering layer, not a feature branch — though a large amount of
user-facing feature work (MAF, alignments grouping, methylation modes, GWAS+LD,
bookmarks-via-session, embedded managed components, autogenerated docs) was built
on top of the new architecture and ships with it.

## Merge strategy (recommendation)

**`origin/main` is fully contained in this branch** — 0 commits on `main` since
the merge-base, because the branch absorbed `main` 68 times. **A merge to `main`
is conflict-free / fast-forwardable.**

**Recommendation: merge the branch whole; do not attempt a partial/selective
merge.** The GPU migration is foundational, not additive: it *removed the
renderer-config layer and collapsed display types* (e.g. Pileup / SNPCoverage /
ReadArcs / ReadCloud → one `LinearAlignmentsDisplay`), and nearly all the
"non-GPU-looking" feature work was built on that new shape. Cherry-picking themes
onto `main` would mean resolving them against an architecture that no longer
exists there, across a DAG with 68 interleaved main-merges. The only cleanly
separable leaves are some `website/` docs and isolated `jbrowse-cli`/`jbrowse-img`
bits — not worth the untangling, and they reference features that only exist here.

Two viable mechanics for merging whole:

| | Squash merge | Single real merge commit |
|---|---|---|
| `main` history | 1 commit | clean via `git log --first-parent`; full history retained |
| Rationale trail | lost from git (but the 30 ADRs in `agent-docs/` preserve the *why* in-tree) | fully preserved |
| `git bisect` going forward | fine | fine + can bisect into the 4 months |
| "history chaos" risk | none | confined to non-first-parent log |

**Squash is reasonable** (matches the stated instinct; the ADRs already preserve
intent in-tree). A **single real merge commit** costs nothing extra since `main`
hasn't diverged and keeps the work bisect-able later. Either way, use this
document as the squash-commit body / PR description.

## Architecture overview

This branch replaces JBrowse 2's per-block HTML-canvas rendering with a GPU
pipeline. The contract for reviewers: **data is fetched in RPC workers and
rendered on the main thread**, and the MST model — not React — owns the render
lifecycle. Read `agent-docs/ARCHITECTURE.md` for the canonical reference;
`agent-docs/PRD.md` for invariants; `agent-docs/CONFIG_PATTERN.md` for the
config flow; `agent-docs/MIGRATION_AUDIT.md` for back-compat.

### Core rendering model

- **Workers fetch, main thread renders.** RPC workers parse files and emit
  geometry as **absolute genomic `uint32`** — no regionStart-relative
  arithmetic crosses the worker boundary. Worker output is the source of truth
  for what gets uploaded to the GPU.
- **HAL with three backends.** A Hardware Abstraction Layer
  (`packages/render-core/src/hal/`) hides the WebGPU/WebGL2 difference behind
  one interface (`uploadBuffer`/`drawPass`/`writeUniforms`/`pruneRegions`/…).
  `createGpuHal` tries **WebGPU → WebGL2 → null**; on `null` the plugin
  constructs its **Canvas2D** backend. Canvas2D is the floor, GPU the
  accelerator. `?renderer=webgpu|webgl|canvas2d` forces a backend.
- **SVG export via Canvas2D.** Export and on-screen Canvas2D share the *same
  pure `drawXxxBlocks` draw functions*, routed through `paintLayer` (raster 2×
  PNG or vector `SvgCanvas`). This kills the old drift-prone SVG-only
  `renderToCtx` path. A small set of permitted JSX-SVG exceptions remain
  (scalebars, bezier-arc overlays, shared React-SVG overlays).
- **Precision:** positions stored as `uint32`, converted to clip space via a
  hi/lo Float32 split so T2T-scale coordinates survive float32 clip space.

### MST-owned render lifecycle

Each GPU display composes **`RenderLifecycleMixin`**
(`packages/render-core`). Its only plugin hook is
`startRenderingBackend(backend)`, whose body is a single
`self.attachRenderingBackend(backend, { upload, render })` call.

- The mixin spawns **two autoruns tied to the model's lifetime**: an `upload`
  autorun (push fresh, never-mutated bytes to the GPU) and a `render` autorun.
  MobX auto-tracks every observable read inside each callback, so the right
  autorun re-fires on change — no manual invalidation, no `useEffect` deps.
- **`canvasDrawn`** flips true only when `render` returns truthy with real
  content (returning `false` skips); `clearAllRpcData` resets it. It feeds
  `MultiRegionDisplayMixin.isReady` and the single `displayPhase` getter
  (`renderError | tooLarge | error | loading | ready`) that the shared
  **`DisplayChrome`** branches on. The chrome calls `useRenderingBackend` in
  exactly one place; React components are thin canvas-mounting bridges.
- Context loss, tab-visibility re-render, and terminal-state overlays all live
  in the mixin/chrome, not in plugins. Observables are written only via
  actions.

### Shader codegen

Production draw shaders are authored once in **`.slang`**, cross-compiled to
WGSL + GLSL ES 3.00 by `packages/shader-tools/src/build-shaders.ts`
(`pnpm gen:shaders`). Codegen emits `<name>.generated.ts` containing source
strings, byte offsets/strides, typed uniform/instance interfaces, a typed
`writeUniforms`/`packInstances`, and the GL attribute layout — all derived from
Slang reflection, so packer/struct drift is impossible by construction.
**Never hand-edit `*.generated.ts`**; CI runs `gen:shaders && git diff
--exit-code`.

### Config-layer collapse and back-compat

The separate **renderer-config layer is gone**: visual settings (`color1`,
`featureHeight`, JEXL callbacks) now live directly on the *display* config
schema, read at the render layer via `readConfigValue` on a plain snapshot
(no MST off-thread). Runtime overrides flow through one `configOverrides`
frozen map (`ConfigOverrideMixin`).

- **Display-type collapse:** Pileup / SNPCoverage / ReadArcs / ReadCloud →
  one **`LinearAlignmentsDisplay`**; renderer sub-configs lifted onto displays.
- **Config-slot collapse:** e.g. canvas `color1/color2/color3/outline` →
  `color/connectorColor/utrColor/outlineColor`.
- **Migration strategy:** `preProcessSnapshot` migrations
  (`migrateBasicSnapshot`, `migrateWiggleSnapshot`, `migrateAlignmentsSnapshot`,
  `migrateOldSettingSnapshots`, `migrateTrackConfig`). The rule:
  **widening is always load-safe; only narrowing, type-change, or rename
  crashes** MST validation. Unknown props are silently ignored (acceptable
  loss). A type/enum change only crashes if the dangerous value was actually
  persisted — defaults stripped in `postProcessSnapshot` never reached disk.
  Relocated display/renderer types keep their registered `name`, so old `type:`
  strings still resolve.

## GPU foundation — HAL, backends, shader codegen

*72 files changed, 5,217 insertions(+), 1,103 deletions(-)*

This area replaced JBrowse 2's per-block HTML-canvas rendering with a unified GPU pipeline built on a Hardware Abstraction Layer (HAL) that runs on WebGPU first, falls back to WebGL2, and finally to a Canvas2D software renderer. The original landing introduced the WebGPU backend and HAL, after which every renderer was funneled through the HAL, shaders were migrated from hand-written GLSL/WGSL to Slang source compiled by a new codegen tool, and the upload/render lifecycle was moved off React components into MST models. By the end the foundation was split into three packages — GPU primitives stayed in `@jbrowse/core/src/gpu` (which shrank to a leaner core), shared rendering primitives moved to the new `@jbrowse/render-core`, and Slang shader codegen moved to the new `@jbrowse/shader-tools` — leaving a much smaller, deduplicated, three-backend foundation.

### Features
- Landed the initial WebGPU backend and HAL abstraction, the foundation the rest of the migration builds on.
- Added a Canvas2D fallback renderer so users without WebGL2/WebGPU still get rendering.
- Unified all GPU renderers behind a single HAL interface and added shader validation as part of the same pass.
- Added a Slang-based shader toolchain: shaders are authored in `.slang` and compiled to generated code, with `TextureBinding` slots auto-derived from Slang reflection.
- Added a software/standalone 2D path via a `getPreparedCanvas2D` helper for non-GPU overlays.

### Fixes
- Fixed numerous WebGPU correctness/validation issues that landed iteratively: MSAA sample-count mismatches causing display crashes, padding instance structs to 16-byte boundaries for Chrome's validator, "buffer destroyed" errors when clicking features, restoring MSAA with proper error scopes for wiggle, and acquiring the WebGPU canvas context only after shader compile.
- Fixed alpha-blending bugs: corrected `blendFuncSeparate` asymmetry and synced WebGL2 and WebGPU blend modes; fixed a black-background bug and an arrow alpha bug.
- Fixed GPU resource leaks and context loss: removed `WebGL2Hal` canvas event listeners on dispose, latched context loss to guard `dispose()` from deleting stale GL objects, and made device-lost listeners persistent with an identity check.
- Fixed byte-alignment errors in `GPUQueue.writeBuffer` (LD heatmap) and widened buffer-map types to `ArrayBufferLike`.
- Fixed reversed-region rendering in wiggle and variant displays, and a high-precision chevron culling bug.
- Fixed WebGL2→Canvas2D fallback messaging and added a `createProgram` null check plus DPR default in the synteny path.

### Refactors / architecture
- Extracted the foundation into three packages: GPU/Canvas2D primitives into `@jbrowse/render-core`, Slang codegen into `@jbrowse/shader-tools` (marked private as a repo-coupled build tool), with core GPU shrinking substantially.
- Migrated all shaders from hand-written GLSL/WGSL to Slang incrementally — alignments (with palette packed as u32 ABGR), HiC, wiggle, dotplot, and variant shaders — sharing `hpmath` and a common `slangPass`.
- Moved the GPU upload/render lifecycle out of React components and into MST models, unifying HiC, LD, and variant-matrix displays around an MST autorun and consolidating display lifecycle into a single mixin utility (plus `FetchMixin` and `GlobalDataDisplayMixin` extractions).
- Unified and simplified the per-region GPU lifecycle: a shared `RegionRegistry`, `syncCanvasSize`, consolidated `RenderBlock` type and `buildRenderBlocks` utility, and `BlockFrame` records replacing scattered uniform/overlay writes.
- Removed dead architecture repeatedly: module-level WeakMap caches replaced with per-HAL `LayoutState`, dropped dead storage-buffer paths, `MockHal.passes`, `RegionMeta` HAL API, unreachable WGSL stride checks, and globals.
- Renamed GPU vocabulary so "Gpu" specifically means WebGL/WebGPU-specific code, and tightened the public API surface and internal mixin names for clearer conceptual naming.
- Extracted shared utilities: HP shader constants into `core/gpu/shaderConstants`, WebGL helpers into `core/gpu/webglUtils`, `quadLocal`/`yPxToClipY`/`snapPixelCenterY` shader helpers, and centralized `devicePixelRatio` access in `getDpr()`.
- Centralized `RenderBlock` and standardized upload caches to typed `Map<number, T>`; typed config-reader returns and volatiles.

### Performance
- Added an `uploadChangedRegions` utility applying per-region GPU upload caching across all display types, plus incremental per-ref-group layout to fix an O(N²) whole-genome upload.
- Batched WebGPU command submissions per frame via a uniform ring buffer, and batched MSAA work into a single render pass.
- Removed a synchronous `gl.getError()` stall from the WebGL2 draw path and dropped a sync `gl.getError` stall elsewhere; cached attribute locations and combined coverage uploads.
- Moved synteny picking from GPU framebuffer readback to CPU.

### Tests
- Improved browser-test reliability for GPU-heavy cases by recycling the browser per test to work around WebGL resource exhaustion.

### Docs
- Documented the derived-region-map pattern and HAL semantics, clarified field names, gated/removed debug logging, and referenced the GPU implementation's test files from inline comments.

## Canvas display plugin (feature glyph rendering)

*147 files changed, 13,409 insertions(+), 6,337 deletions(-)*

The canvas plugin was rebuilt from HTML-canvas block rendering into a GPU-instanced feature renderer. Previously features were drawn block-by-block on a 2D canvas with layout computed in workers and shipped as region-relative records; now glyphs (rectangles, lines, chevrons, arrows, gene/CDS, peptide overlays) are encoded as instanced GPU geometry rendered through a WebGPU → WebGL2 → Canvas2D fallback chain, with workers emitting absolute genomic uint32 layout and the main-thread MST model owning upload/render lifecycle and hit-testing via flatbush indexes. Along the way `LinearFeatureDisplay` was consolidated into a `LinearBasicDisplay` (canvas base + gene layer) family, config was modernized to typed overrides, and the whole display family was unified under a shared `DisplayChrome`/GPU-lifecycle architecture.

### Features
- New GPU-instanced glyph renderer for all feature types — rectangles, lines, chevrons, arrows — with dynamic zoom-driven chevrons, hit detection, and hover highlighting (landed incrementally across the early WebGL/WebGPU bring-up).
- Gene glyph rendering with exon/CDS/UTR structure, directional chevrons, and inline peptide (amino-acid) letter translation drawn as a GPU overlay; CDS regions deduped and translated inline for speed.
- `MultiLinearBasicDisplay` for rendering features in per-sample lanes.
- Super-compact and compact display modes, with a view-level `setCompactness` action and a "Compact all tracks" menu item; compact/super-compact scaling moved to the main thread to skip RPC re-fetch.
- Collapse-introns support, including a `CollapseIntronsDialog` (consolidated to live in the LGV/canvas plugin).
- "Group by" functionality and a `showOnlyGenes` / glyph-mode type filter (auto/all/longestCoding).
- Auto density-gated `showLabels`/`showDescriptions`: labels are decimated rather than fully hidden in collapsed mode, descriptions are clickable, and the auto-hide threshold became configurable.
- Auto-fit-height menu toggle with a scrollbar-aware overflow indicator, plus a single consolidated overflow-expand IconButton.
- `fetchFullFeature` / `fetchFullFeature` action on the base canvas display and a feature-outline display option.
- SVG export path for canvas glyphs, clipped to the visible region via a shared `SvgClipRect` helper and routed through a shared `drawXxxToCtx` pattern.

### Fixes
- Reversed-region rendering corrected end-to-end: flip applied at output rather than in coordinate math, eliminating double-flips across canvas, wiggle, variant, and synteny backends and in SVG export.
- Blank-region bugs on pan-back and on `displayedRegion` change fixed by keeping `loadedRegions` in sync with the RPC data map and adding extra clearing.
- Hover-highlight fixes: highlight only expands horizontally (not vertically), flicker from a relayout autorun loop removed, and hover rects now also show when mousing over feature labels.
- Label/feature desync and layout overlap fixed; feature descriptions truncated by width and array-valued labels handled; description label width capped (`MAX_DESCRIPTION_LABEL_WIDTH_PX = 200`) in `rpcTypes`, flowing into layout row-packing and flatbush hit boxes to remove phantom tooltips over empty regions.
- Hit-testing fixes: multi-region scan no longer stops after the first miss, hit-test boundary and arrow filter corrected, and `preventDefault` on context menu only fires when a feature is actually hit.
- Peptide/codon readability fixes: lighter codon backgrounds, per-codon contrast-aware text color so black/white letters stay legible over alternating backgrounds, and protein-letter overlay cleanup.
- Chevron rendering fixes: high-precision culling, inner-vertex math in WGSL, fwidth-based AA edge fade, thinner origin-chevron styling, and arrow alpha bug.
- WebGPU validation fix: WGSL instance structs padded to 16-byte boundaries for Chrome.
- Render correctness: canvas no longer returns "drawn" before any data exists; loading overlay shown immediately on track open and kept honest while stale data is on screen; banner flicker on small zoom fixed by deriving `regionTooLarge`.
- Migration fixes: `showLabels` enum migration, `color1/2/3` → `color`/`connectorColor`/`utrColor` rename with legacy mapping, `outline` → `outlineColor`, and various `preProcessSnapshot` bug fixes.
- Subfeature labels in "below" mode now allocate vertical space in the gene glyph; description collapses up when labels are hidden, avoiding scroll jumps on relayout.
- `setFeatureDensityStatsLimit`, force-load gate, scroll-sync leak, and ctrl+scroll vertical-scroll bugs fixed.

## Core package (config, MST, RPC, UI, util) — excluding src/gpu

*288 files changed, 9,457 insertions(+), 7,607 deletions(-)*

The GPU migration reshaped core from a renderer-registry / server-side-canvas world into a thin, typed substrate that feeds the new GPU pipeline. The old `CoreRender` RPC, the renderer registry, and the server-side renderer/canvas classes were deleted; config slots collapsed from per-slot sub-models into bare value props with default-stripping via `types.stripDefault`; RPC was rebuilt around `RpcClient`/`RpcServer` with absolute-coordinate worker output; and the UI layer was heavily de-duplicated, modernized to React 19 (dropping `forwardRef`/polyfills), and pulled shared dialogs, fetch hooks, and overlays into core. The net result is a smaller, more strongly typed core with far fewer casts and dead paths.

### Features
- Track config `displays` object shorthand: a track may declare displays as an object map instead of an explicit array, expanded by `expandTrackConfigShorthand`; documented in the autogen reference.
- Config override system: added `ConfigOverrideMixin`, `getConfWithOverride`, `getConfSnapshot`, and `readConfigValue` in core, unifying how displays read settings that override config-slot defaults (`<name>Override` + resolved getter); migrated remaining `getConf(self)` call sites and removed the `CachedConfig` indirection.
- New theme palette colors: first-class `N`-base color used everywhere N appears, and a `textHighlight` color wired into search-highlight hooks; consolidated theme overrides and made a serializable theme shippable to workers.
- "All tracks" extensible group menu generalized from the old "Compact all tracks", plus a "Copy and open track" action alongside "Copy track".
- Add-track workflow: intent-led "Add…" labels via `displayName` and a category-grouped Add Track dropdown.
- About widget: surfaces git commit hash separately from version and enriches stack-trace environment info; added `CopyToClipboardButton`.
- New core utilities: `truncateMiddle`, `makeTrackId`, `openTabixIndexFilehandle`, `makeFeatureIntervalTreeMap`, `getAdapterId`, `fetchJson`, `useFetch`, `cssColorToRgb`/`setAbgrFill`, and a shared `PluggableComponent` helper for extension points.
- UI components promoted to core: shared `BaseExportSvgDialog` (replacing 5 plugin copies), `FileDropZone` (dedup of four dropzones), `WidgetHeader`, `ActionLink`/`LinkMenuRow`, `ErrorBanner` with a stack-trace button and wrapping text, `SubmitDialog` wrapper, and a centralized tooltip mechanism; menus now open below the trigger icon.

### Fixes
- Invalid/broken track configs are now surfaced as snackbar errors instead of crashing the app, routed through a single `notifyError`; unhydratable open tracks are dropped at session load and tolerated at restore/render.
- RPC robustness: tolerate non-cloneable args, dedup ephemeral auth accounts by origin in the auth-retry path, fix worker-count cap and circular-depth bugs, and guard method dispatch against prototype-chain lookups.
