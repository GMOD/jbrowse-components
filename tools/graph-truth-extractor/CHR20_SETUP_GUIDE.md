# CHR20 GetSubgraph Validation Setup Guide

This guide explains how to set up and run the GetSubgraph RPC validation tests
on the chr20_region pangenome graph.

## Quick Start

### One-Command Setup

```bash
# Using Node.js
node --experimental-strip-types tools/graph-truth-extractor/setup-chr20-validation.ts

# Or using bash
bash tools/graph-truth-extractor/setup-chr20-validation.sh
```

This will:

1. Verify the chr20_region.gfa file exists
2. Generate tabix indices (if needed)
3. Run all 6 validation tests

## What Gets Generated

The setup script generates the following **gitignored** index files:

```
test_data/chr20_region.pos.bed.gz          - Position index (~4.3 KB)
test_data/chr20_region.pos.bed.gz.tbi      - Tabix index for positions
test_data/chr20_region.synteny.bed.gz      - Synteny mappings (~368 B)
test_data/chr20_region.synteny.bed.gz.tbi  - Tabix index for synteny
test_data/chr20_region.edges.spatial.bed.gz - Edge spatial index (~3.7 KB)
test_data/chr20_region.edges.spatial.bed.gz.tbi - Tabix index for edges
test_data/chr20_region.seglens.bin         - Segment lengths (~1.1 KB)
test_data/chr20_region.gfa.truth-cache/    - vg index cache (3.6 GB)
```

All of these are listed in `.gitignore` to avoid committing large files.

## Validation Tests

The test suite (`getSubgraph-validation.test.ts`) includes 6 tests:

1. **getSubgraph_chr20_small_region_produces_segments**
   - Tests small region (0-10,000 bp)
   - Verifies 199 segments and 428 edges produced
   - Checks GFA 1.1 format compliance

2. **getSubgraph_chr20_medium_region_produces_graph_structure**
   - Tests medium region (10,000-50,000 bp)
   - Verifies graph structure production
   - Validates >0 segments returned

3. **getSubgraph_chr20_edges_reference_valid_segments**
   - Tests edge referential integrity
   - Verifies all edge endpoints reference valid segments
   - Tests 50,000 bp region

4. **getSubgraph_chr20_walks_reference_valid_segments**
   - Tests walk node validity
   - Verifies all walk nodes are in segment set
   - Tests 100,000 bp region

5. **getSubgraph_chr20_large_region_produces_many_segments**
   - Tests scaling to large regions (0-200,000 bp)
   - Verifies >50 segments produced
   - Validates linear scaling behavior

6. **getSubgraph_chr20_empty_region_returns_header**
   - Tests edge case handling
   - Verifies correct GFA header response for out-of-bounds regions

## Test Execution

Run tests directly with:

```bash
# All validation tests
npm test -- plugins/comparative-adapters/src/GfaTabixAdapter/__tests__/getSubgraph-validation.test.ts

# With verbose output
npm test -- plugins/comparative-adapters/src/GfaTabixAdapter/__tests__/getSubgraph-validation.test.ts --verbose

# Watch mode
npm test -- plugins/comparative-adapters/src/GfaTabixAdapter/__tests__/getSubgraph-validation.test.ts --watch
```

## Manual Index Generation

If you only want to generate indices without running tests:

```bash
# Using gfa-to-tabix directly
tools/gfa-to-tabix/target/release/gfa-to-tabix \
  test_data/chr20_region.gfa \
  test_data/chr20_region
```

## Requirements

- **GFA file:** `test_data/chr20_region.gfa` (must exist)
  - 278 segments, 90 haplotype paths
  - Generated by graph-truth-extractor from real pangenome data
- **gfa-to-tabix tool:** Built at
  `tools/gfa-to-tabix/target/release/gfa-to-tabix`
  - Build with: `cd tools/gfa-to-tabix && cargo build --release`

## Troubleshooting

### "chr20_region.gfa not found"

The GFA file should be generated separately. Contact the repository maintainers
for the data file.

### "gfa-to-tabix not built"

Build the tool:

```bash
cd tools/gfa-to-tabix
cargo build --release
```

### "Tabix index file not found during test"

Re-run the setup script to regenerate indices:

```bash
node --experimental-strip-types tools/graph-truth-extractor/setup-chr20-validation.ts
```

### Tests fail with "ENOENT" errors

Ensure you're running from the repository root:

```bash
cd /path/to/jbrowse-components
npm test -- plugins/comparative-adapters/src/GfaTabixAdapter/__tests__/getSubgraph-validation.test.ts
```

## CI Integration

To integrate validation into CI:

```yaml
# .github/workflows/validate-getsubgraph.yml
name: GetSubgraph Validation
on: [push, pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm ci
      - run: cargo build --release -C tools/gfa-to-tabix
      - run:
          node --experimental-strip-types
          tools/graph-truth-extractor/setup-chr20-validation.ts
```

## Performance Notes

- **Index generation:** ~2-3 seconds for chr20_region.gfa
- **Test execution:** ~3.5 seconds for full suite
- **Total setup time:** ~6 seconds (first time only)
- **Subsequent runs:** ~3.5 seconds (indices cached)

Index files are ~10 KB total (easily cacheable in CI).
