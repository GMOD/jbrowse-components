`@jbrowse/react-circular-genome-view2` embeds an interactive **circular** genome
view in any React app. It shines for structural variants, chromosome-scale
rearrangements, and whole-genome summaries, where chromosome-spanning breakends
become visible chord-style across the circle. If you need a linear,
track-by-track view instead, see
[`@jbrowse/react-linear-genome-view2`](https://jbrowse.org/jb2/docs/embedded_components/).

## A minimal embed

`createViewState` builds the MobX-state-tree model — wrap it in
`useState(() => …)` so it survives parent re-renders rather than being rebuilt
each time. The CGV config reuses the JBrowse format, with one difference: it
takes a single `assembly` rather than an `assemblies` array. Use the `init`
shorthand on `defaultSession.view` to declare which assembly and tracks open on
first paint:

```jsx
import {
  JBrowseCircularGenomeView,
  createViewState,
} from '@jbrowse/react-circular-genome-view2'

const [state] = useState(() =>
  createViewState({
    assembly,
    tracks,
    defaultSession: {
      name: 'My session',
      view: {
        type: 'CircularView',
        init: { assembly: 'volvox', tracks: ['volvox_sv_test'] },
      },
    },
  }),
)
return <JBrowseCircularGenomeView viewState={state} />
```

The lower-ceremony [managed](../managed/) `<CircularGenomeView>` takes the same
data as plain props and owns the engine for you.

## Configuration reference

The autogenerated schema references list every available field: track/assembly
slots under [docs/config](https://jbrowse.org/jb2/docs/config/) and view
snapshot properties under [docs/models](https://jbrowse.org/jb2/docs/models/)
(e.g. [CircularView](https://jbrowse.org/jb2/docs/models/circularview/)). To
open a track in response to a runtime event rather than at first paint, call
`showTrack` imperatively — see [show a track](../show-track/).
