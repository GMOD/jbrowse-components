# @jbrowse/react-circular-genome-view2

This is the [Storybook](https://storybook.js.org/) for
[`@jbrowse/react-circular-genome-view2`](https://www.npmjs.com/package/@jbrowse/react-circular-genome-view2),
a React component that embeds an interactive **circular** genome view in any
React application — useful for showing structural variants, chromosome-scale
rearrangements, or whole-genome summaries.

If you need a linear track-by-track view instead, see
[`@jbrowse/react-linear-genome-view2`](https://www.npmjs.com/package/@jbrowse/react-linear-genome-view2).

## Configuration shape

The CGV component reuses the JBrowse config format, with one difference: it
takes a single `assembly` rather than an `assemblies` array. Use the `init`
shorthand on `defaultSession.view` to declare which assembly and tracks open on
first paint:

```js
const state = createViewState({
  assembly: { /* assembly config */ },
  tracks: [ /* track configs */ ],
  defaultSession: {
    name: 'My session',
    view: {
      type: 'CircularView',
      init: {
        assembly: 'hg38',
        tracks: ['my-sv-track-id'],
      },
    },
  },
})
```

The component can be configured inline (JS objects in your source) or from a
standalone JSON file —
[example](https://github.com/GMOD/jbrowse-components/blob/HEAD/products/jbrowse-react-circular-genome-view/docs/example.md).

When hand-authoring, the autogenerated schema references list every available
field: track/assembly slots under
[docs/config](https://jbrowse.org/jb2/docs/config/) and view snapshot properties
under [docs/models](https://jbrowse.org/jb2/docs/models/) (e.g.
[CircularView](https://jbrowse.org/jb2/docs/models/circularview/)).

## Opening tracks at runtime

For tracks that open in response to runtime conditions (a button click, a search
result), call `state.session.view.showTrack(trackId)` imperatively after
construction instead of baking the track into `defaultSession`:
