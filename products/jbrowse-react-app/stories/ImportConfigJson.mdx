# Importing config.json

JBrowse Web auto-loads a `config.json` from the current directory (or
the `?config=` URL parameter). The embedded `react-app2` component
does **not** — it makes no assumptions about URLs and lets you control
how and when the config is loaded. This is typical of low-level
components: more flexibility, slightly more code.

There are two common patterns:

## Static import

If your config is part of your app bundle, use a regular ES import.
Bundlers handle the JSON natively, and the config ships alongside your
JavaScript — no runtime fetch.

<Story of={WithImportConfigJson} />

## Runtime fetch

If the config lives on a server (or differs per environment, per user,
per route), fetch it before constructing `createViewState`. Wrap
`viewState` in `useState`/`useEffect` so React can render once the
fetch resolves.

<Story of={WithFetchConfigJson} />

Use static import for shipped configurations and runtime fetch for
configs that are loaded on demand or generated server-side.
