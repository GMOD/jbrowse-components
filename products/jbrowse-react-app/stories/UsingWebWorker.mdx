# Web Worker RPC

Using WebWorker RPC reduces main-thread blocking when browsing BAM/CRAM data.
Import `makeWorkerInstance` and pass it to `createViewState`:

```js
import { createViewState, JBrowseApp } from '@jbrowse/react-app2'
import makeWorkerInstance from '@jbrowse/react-app2/esm/makeWorkerInstance'

const state = createViewState({
  config: {
    ...config,
    configuration: { rpc: { defaultDriver: 'WebWorkerRpcDriver' } },
  },
  makeWorkerInstance,
})
```

## Bundler notes

**Webpack** — set `output.publicPath: 'auto'` so the worker resolves its own
URL. See
[this comment](https://github.com/webpack/webpack/issues/13791#issuecomment-897579223)
and the [official guide](https://webpack.js.org/guides/web-workers/).

**Vite** — the worker is constructed via
`new Worker(new URL(...), { type: 'module' })`, which Vite handles natively.

## Configuration reference

The `rpc` config block (driver selection, worker count, timeouts) is documented
in the autogenerated
[WebWorkerRpcDriver](https://jbrowse.org/jb2/docs/config/webworkerrpcdriver/)
and [BaseRpcDriver](https://jbrowse.org/jb2/docs/config/baserpcdriver/)
references.
