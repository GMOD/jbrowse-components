# Web worker RPC

By default the embedded app runs all data parsing and rendering on the main
thread, which can stall the UI on large alignments datasets. The WebWorker RPC
moves that work off the main thread, dramatically reducing hitches when
scrolling BAM/CRAM tracks.

To enable it, import `makeWorkerInstance`, pass it to `createViewState`, and set
`WebWorkerRpcDriver` as the default RPC driver in the configuration block.

A working integration with create-react-app is at
[jbrowse.org/jb2/docs/embedded_components](https://jbrowse.org/jb2/docs/embedded_components/).

## Bundler notes

**Webpack** — set `output.publicPath: 'auto'` so the worker resolves its own URL
correctly. See
[this comment](https://github.com/webpack/webpack/issues/13791#issuecomment-897579223)
and the [official guide](https://webpack.js.org/guides/web-workers/).

**Other bundlers** — the worker is constructed via
`new Worker(new URL(...), { type: 'module' })`, which Vite handles natively.

The web worker is not enabled by default precisely because of these bundler
requirements, but we recommend turning it on whenever your toolchain supports
it.
