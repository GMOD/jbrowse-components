# Welcome

This is the [Storybook](https://storybook.js.org/) for
[`@jbrowse/react-app2`](https://www.npmjs.com/package/@jbrowse/react-app2),
which embeds the **full JBrowse 2 application** ‚Äî menu bar, view manager, drawer
widgets, and all view types ‚Äî inside any React component tree.

## App vs Linear Genome View ‚Äî which to use?

JBrowse ships two embeddable React components. They serve different use cases:

- **`@jbrowse/react-app2`** is the whole app. You get every view type (linear,
  circular, dotplot, synteny, spreadsheet), the global menu bar, drag-to-resize
  panels, the import wizard, and a full session model with tabs. Pick this when
  you're building a _genome browser_ that happens to live in your page.
- **[`@jbrowse/react-linear-genome-view2`](?path=/docs/getting-started--docs)**
  is just the linear track view, no app chrome. Pick this when you want a _track
  viewer_ embedded inside a larger application that owns the layout, navigation,
  and surrounding UI.

If you're not sure, start with the LGV component ‚Äî it's lighter weight and
easier to integrate. Move to react-app when you need the app-level features.

## Configuration shape

`react-app2` accepts the same configuration format as JBrowse Web ‚Äî this is what
makes round-tripping configs between the two trivial:

```js
const state = createViewState({
  config: {
    configuration: {
      /* global app config (theme, RPC driver, etc.) */
    },
    assemblies: [
      {
        name: 'hg38',
        sequence: {
          /* ... */
        },
      },
    ],
    tracks: [{ trackId: 'my_track', type: 'FeatureTrack' /* ... */ }],
    defaultSession: {
      views: [
        {
          type: 'LinearGenomeView',
          init: {
            assembly: 'hg38',
            loc: 'chr1:1,000,000-1,100,000',
            tracks: ['my_track'],
            tracklist: true, // open track selector on load
          },
        },
      ],
    },
  },
})
```

Unlike JBrowse Web, the embedded component does **not** automatically fetch a
`config.json` based on URL parameters. If you want that behavior, do the fetch
yourself and pass the result to `createViewState` ‚Äî see
[Importing config.json](?path=/docs/importing-config-json-files--docs).
