## An overview of the @jbrowse/react-app2 API

The @jbrowse/react-app2 component relies on the same
[configuration](https://jbrowse.org/jb2/docs/config_guide) as the main JB2 app.

The `createViewState` constructor accepts a config object in the same format as a
`config.json` for `@jbrowse/web`. Unlike `@jbrowse/web`, it does not fetch
`config.json` automatically; you must fetch it manually if needed.

Config structure:

```json
{
  "configuration": {
    /* global configs here, e.g. theme */
  },
  "assemblies": [
    /* list of assembly configurations */
  ],
  "tracks": [
    /* track configurations */
  ],
  "defaultSession": {
    /* optional: which views and tracks to open on load */
  }
}
```

## Opening a location on startup with `init`

The `defaultSession.views[*].init` shorthand is the recommended way to open a
specific location with tracks on startup. It avoids constructing the verbose
internal session format:

```js
defaultSession: {
  name: 'My session',
  views: [{
    id: 'view1',
    type: 'LinearGenomeView',
    init: {
      assembly: 'hg38',
      loc: 'chr1:1,000,000-1,100,000',
      tracks: ['my_genes_track'],
    },
  }],
},
```

This works for all view types: `LinearSyntenyView`, `DotplotView`, etc., with
their own `init` fields (see the [Synteny Views](?path=/docs/synteny-views--docs)
story for an example).
