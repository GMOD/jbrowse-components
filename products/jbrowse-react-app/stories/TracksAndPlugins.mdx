# Tracks and plugins

Two ways to extend a running view: register additional tracks at runtime, and
load plugins that add new track types, adapters, renderers, or menu items.

## Adding tracks programmatically

Call `addTrackConf` and `showTrack` from an event handler — not during render.
The slots available on a track config come from its
[config docs](https://jbrowse.org/jb2/docs/config/) (e.g.
[FeatureTrack](https://jbrowse.org/jb2/docs/config/featuretrack/)), and each
`adapter` type has its own page too (e.g.
[Gff3TabixAdapter](https://jbrowse.org/jb2/docs/config/gff3tabixadapter/)):

```js
state.jbrowse.addTrackConf({
  type: 'FeatureTrack',
  trackId: 'my_genes',
  name: 'My Genes',
  assemblyNames: ['hg38'],
  adapter: {
    type: 'Gff3TabixAdapter',
    uri: 'https://example.com/genes.gff.gz',
  },
})
state.session.views[0]?.showTrack('my_genes')
```

## Using plugins

Pass plugin classes directly to `createViewState`:

```js
import Plugin from '@jbrowse/core/Plugin'

class MyPlugin extends Plugin {
  name = 'MyPlugin'
  install(pluginManager) {
    /* add track types, extension points, etc. */
  }
  configure() {}
}

const state = createViewState({ config, plugins: [MyPlugin] })
```

For NPM-published plugins, import and pass the class the same way:

```js
import GWASPlugin from 'jbrowse-plugin-gwas'
const state = createViewState({ config, plugins: [GWASPlugin] })
```

### Loading a plugin from a URL at runtime

If you don't want to bundle a plugin, fetch it at runtime with `loadPlugins`,
which returns `{ plugin, pluginMetadata }[]`, and pass the plugin classes to
`createViewState`:

```js
import { loadPlugins, createViewState } from '@jbrowse/react-app2'

const plugins = await loadPlugins([
  {
    name: 'UCSC',
    url: 'https://unpkg.com/jbrowse-plugin-ucsc/dist/jbrowse-plugin-ucsc.umd.production.min.js',
  },
])
const state = createViewState({ config, plugins: plugins.map(p => p.plugin) })
```
