A richer, real-world hg38 session showing off the kinds of tracks a production
embed carries at once:

- **NCBI RefSeq genes** (GFF3+tabix) with a `TrixTextSearchAdapter`, so typing a
  gene symbol in the location box jumps the view to it
- **Repeats** from a BigBed
- **NA12878 exome** alignments (CRAM) from the 1000 Genomes project
- **1000 Genomes variant calls** (VCF+tabix)
- **phyloP100way conservation** as a quantitative (BigWig) track

Every file is fetched directly over HTTP range requests from public buckets —
no server-side component is required. The `defaultSession` opens a linear genome
view at a locus on chromosome 1 with the exome and conservation tracks showing.

This is the same `createViewState({ config })` flow as the
[basic example](../basic-example/), just with a fuller `config`. On heavy
alignments data like the CRAM track here, enable the
[web worker RPC](../with-web-worker/) to keep parsing off the main thread. The
config slots used are documented in the autogenerated reference:
[AlignmentsTrack](https://jbrowse.org/jb2/docs/config/alignmentstrack/) /
[CramAdapter](https://jbrowse.org/jb2/docs/config/cramadapter/),
[VariantTrack](https://jbrowse.org/jb2/docs/config/varianttrack/) /
[VcfTabixAdapter](https://jbrowse.org/jb2/docs/config/vcftabixadapter/),
[QuantitativeTrack](https://jbrowse.org/jb2/docs/config/quantitativetrack/) /
[BigWigAdapter](https://jbrowse.org/jb2/docs/config/bigwigadapter/), and
[TrixTextSearchAdapter](https://jbrowse.org/jb2/docs/config/trixtextsearchadapter/).
