SARS-CoV-2 (29,903 bp) built from a single config object bundled with the app.
The gene annotations are inline (a `FromConfigAdapter` `FeatureTrack`), while the
two bulky pieces — the reference sequence and the per-position diversity track —
are small standard flatfiles hosted on `jbrowse.org/demos`, so the committed
config stays a few KB instead of megabytes of inlined data:

- reference: `IndexedFastaAdapter` (`covid.fa` + `.fai`)
- diversity (Shannon-entropy): `BigWigAdapter` (`covid_entropy.bw`)

The config is generated by `scripts/gen-nextstrain-demos.mjs` from the live
[Nextstrain SARS-CoV-2 dataset](https://nextstrain.org/ncov/open/global/6m): the
reference is the build's published sequence, gene coordinates come from the
dataset's `genome_annotations`, and the diversity track is reconstructed per
position from the phylogeny's nucleotide mutations. The same generator produces
the [Zika](../nextstrain-zika/), [Ebola](../nextstrain-ebola/),
[measles](../nextstrain-measles/), and [RSV-A](../nextstrain-rsv-a/) demos.

Because the whole config is a plain JS object, it can be imported, generated, or
templated by your own code. See [default session](../default-session/) for the
session structure and
[the config guide](https://jbrowse.org/jb2/docs/config_guide) for the
track/assembly shapes.
