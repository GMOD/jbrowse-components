Browse the [Pan-UK Biobank](https://pan.ukbb.broadinstitute.org/) GWAS across
its full catalog of ~7,200 phenotypes. The search box filters a trimmed copy of
the official Pan-UKBB phenotype manifest (generated by
`scripts/gen-panukb-manifest.mjs`). Picking a phenotype loads that trait's
tabix-indexed summary-statistics file straight from the Pan-UKBB public S3
bucket and renders it as a Manhattan plot. The **Population** dropdown switches
which column drives the plot, either the cross-ancestry meta-analysis or any
single analyzed ancestry (European, African, Admixed American, Central/South
Asian, East Asian, Middle Eastern), and only offers the populations that trait
was actually run in.

Each phenotype file is one `GWASTrack` with a `GWASAdapter`. The Pan-UKBB flat
files expose `neglog10_pval_*` columns that are already −log₁₀(p), so the
adapter reads the selected population column directly as the Manhattan score
(`scoreTransform: 'none'`, the default). For GWAS files whose p-value column is
**not** pre-transformed, `GWASAdapter` offers a `scoreTransform` slot:
`negLog10` for a raw p-value, or `negLog10FromLn` for a natural-log p-value (the
convention Pan-UKBB uses in its Hail-format release, "ln P, not −log₁₀P"). The
transform is applied natively per feature, so it stays fast across the millions
of variants in a genome-wide view.

Featured phenotypes (height, BMI, LDL, type 2 diabetes, coronary disease,
asthma, …) open zoomed to a known lead locus. Any other phenotype opens on
chromosome 1, and you can navigate or search to any gene. GWAS rendering comes
from the GWAS plugin, a core plugin of `@jbrowse/react-linear-genome-view2`, so
no runtime plugin loading is needed. For LD-colored Manhattan plots, see
[LocusZoom-style LD](../locus-zoom-ld/).

See the
[GWAS track guide](https://jbrowse.org/jb2/docs/config_guides/gwas_track/) and
the [GWASTrack](https://jbrowse.org/jb2/docs/config/gwastrack/) /
[GWASAdapter](https://jbrowse.org/jb2/docs/config/gwasadapter/) /
[LinearManhattanDisplay](https://jbrowse.org/jb2/docs/config/linearmanhattandisplay/)
config reference for the underlying track setup.
