# Initialization

This page covers the ways to set up a JBrowse view at startup, and when each
form fits. The shortest path ‚Äî and what Getting Started recommends ‚Äî is the
`init` field on `defaultSession.view`.

## The `init` shape

```js
init: {
  assembly: 'hg38',                  // assembly name or alias
  loc: 'chr1:11,106,077-11,261,675', // any locstring
  tracks: ['my-gene-track-id'],       // trackIds to open
}
```

This is the same `init` shape that JBrowse Web serializes into its
`?session=spec-‚Ä¶` URL parameter ‚Äî see the
[URL query parameter API](https://jbrowse.org/jb2/docs/urlparams/), which
documents the wire form of these same fields. The embedded component makes no
assumptions about URLs, so you pass `init` directly.

Sibling fields on `defaultSession.view` configure the view itself ‚Äî
`showCytobandsSetting`, `showGridlines`, `colorByCDS`, etc. A complete example
using cytobands, ref-name aliases, and a tabix-indexed RefSeq gene track:

## Advanced `init` (per-track snapshots, tracklist, nav, highlight)

`init` accepts more than trackId strings. Each `tracks` entry can be an object
carrying a `displaySnapshot` (initial display state ‚Äî type, height, score range,
color scheme) or a `trackSnapshot` (e.g. `pinned: true`), and the view itself
can be configured declaratively with `tracklist`, `nav`, and `highlight`. This
is the embedded equivalent of the session-spec "advanced track configuration"
used in JBrowse Web URL params.

```js
init: {
  assembly: 'hg38',
  loc: 'chr1:11,106,077-11,261,675',
  tracklist: true,                 // open the track selector on load
  nav: true,                       // keep the navigation bar visible
  highlight: ['chr1:11,170,000-11,190,000'],
  tracks: [
    { trackId: 'my-gene-track-id', displaySnapshot: { height: 200 } },
  ],
}
```

### Alignments track example

`displaySnapshot` is most useful on alignments tracks, where the display has a
large configuration surface. This opens a CRAM track colored by pair
orientation, with soft-clipping revealed and an enlarged height ‚Äî all
declaratively, without any post-mount imperative calls:

```js
init: {
  assembly: 'GRCh38',
  loc: '1:100,987,200..100,987,450',
  tracks: [
    {
      trackId: 'my-cram-track-id',
      displaySnapshot: {
        type: 'LinearAlignmentsDisplay',
        height: 250,
        showSoftClipping: true,
        colorBySetting: { type: 'pairOrientation' },
      },
    },
  ],
}
```

## Top-level `location` (no `defaultSession`)

If you don't need a full default session ‚Äî just "land on this region" ‚Äî pass a
top-level `location` to `createViewState`. It accepts either a locstring or a
`{ refName, start, end }` object. The object form is useful when you already
have structured coordinates and don't want to format and reparse a string:

```js
const state = createViewState({
  assembly,
  tracks,
  location: { refName: '10', start: 29838737, end: 29838819 },
})
```

## The full snapshot form

`init` is shorthand for what would otherwise be a verbose snapshot under
`defaultSession.view.tracks`. The verbose form exists for one reason: it's
exactly what gets serialized when you save or export a session, so it can
round-trip arbitrary view state. Reach for it only when you are loading a
session you previously exported (see [Default sessions](#default-sessions)
below).

Both forms below open the same RefSeq genes track at the same locus on GRCh38.

### With `init` (recommended)

```js
defaultSession: {
  name: 'My session',
  view: {
    type: 'LinearGenomeView',
    init: {
      assembly: 'hg38',
      loc: 'chr1:11,106,077-11,261,675',
      tracks: [
        'ncbi-refseq-genes',
      ],
    },
  },
}
```

### Equivalent full snapshot

```js
defaultSession: {
  name: 'My session',
  view: {
    id: 'linearGenomeView',
    type: 'LinearGenomeView',
    bpPerPx: 188.34,
    offsetPx: 156721,
    displayedRegions: [
      {
        refName: 'chr1',
        start: 0,
        end: 248956422,
        assemblyName: 'hg38',
      },
    ],
    tracks: [
      {
        type: 'FeatureTrack',
        configuration: 'ncbi-refseq-genes',
        displays: [
          {
            type: 'LinearBasicDisplay',
            configuration: 'ncbi-refseq-genes-LinearBasicDisplay',
          },
        ],
      },
    ],
  },
}
```

IDs at every level, plus precomputed `bpPerPx` / `offsetPx` instead of a
locstring. Use this only for round-tripping an exported session.

## Default sessions

A _session_ is JBrowse's runtime representation of "what's open" ‚Äî which views,
which tracks, which display settings. Passing `defaultSession` to
`createViewState` restores that state on first paint. For most use cases ‚Äî the
declarative `init` field above is easier to author by hand. Reach for the full
`defaultSession` when you need finer control over per-track display settings, or
when you're round-tripping a session exported from JBrowse Web.

### Anatomy of a session

```js
const defaultSession = {
  name: 'My session',
  view: {
    id: 'linearGenomeView',
    type: 'LinearGenomeView',
    tracks: [
      // open tracks, in render order
    ],
  },
}
```

Each entry in `tracks` references a track configuration plus the display(s) you
want active:

```js
{
  type: 'ReferenceSequenceTrack',
  configuration: 'GRCh38-ReferenceSequenceTrack',
  displays: [
    {
      type: 'LinearReferenceSequenceDisplay',
      configuration:
        'GRCh38-ReferenceSequenceTrack-LinearReferenceSequenceDisplay',
    },
  ],
}
```

The `configuration` strings are the `trackId` (or `displayId`) of the matching
entry in `tracks` / its `displays` array.

### Exporting from JBrowse Web

The fastest way to get a working `defaultSession` is to build the view
graphically in the main JBrowse Web app and export it:

- Open a single linear genome view in JBrowse Web with the tracks and settings
  you want.
- Use **File ‚Üí Export session** to download a `session.json`.
- Drop the file into your app and pull out the view:

```js
import session from './session.json'

const defaultSession = {
  name: 'My session',
  view: session.session.views[0],
}
```

This avoids hand-authoring the deeply-nested track/display tree.

## Configuration shape

The component reuses the same
[configuration](https://jbrowse.org/jb2/docs/config_guide) format as JBrowse
Web, with two differences:

**Single assembly**, not an array. JBrowse Web takes `assemblies: [...]`; the
embedded LGV takes one `assembly: {...}`.

**Inline JS or external JSON**. The config can be a plain JS object built in
your app, or loaded from a standalone JSON file fetched at runtime.

## Schema reference for hand-authoring

When you hand-write a `defaultSession.view` snapshot or a track `configuration`,
the exact set of available fields comes straight from JBrowse's models. Two
autogenerated references list every property, getter, action, and config slot ‚Äî
they're the authoritative answer to "what can I put here?":

- **State model docs** describe the `view` snapshot you put under
  `defaultSession`. The view itself is the
  [LinearGenomeView state model](https://jbrowse.org/jb2/docs/models/lineargenomeview/)
  ‚Äî its `#property` entries (`displayedRegions`, `bpPerPx`, `offsetPx`,
  `tracks`, `showGridlines`, ‚Ä¶) are exactly the snapshot keys shown above. Each
  display has its own model too, e.g.
  [LinearReferenceSequenceDisplay](https://jbrowse.org/jb2/docs/models/linearreferencesequencedisplay/).
- **Config docs** describe each `tracks[]`/`assembly` entry. The slots on, e.g.,
  [FeatureTrack](https://jbrowse.org/jb2/docs/config/featuretrack/),
  [AlignmentsTrack](https://jbrowse.org/jb2/docs/config/alignmentstrack/), and
  [BaseAssembly](https://jbrowse.org/jb2/docs/config/baseassembly/) are the keys
  you set on a track or assembly config. Browse the full set under
  [docs/config](https://jbrowse.org/jb2/docs/config/).

A practical loop: build the view you want in JBrowse Web, **Export session** to
see the concrete snapshot, then use these references to understand and trim each
field by hand.

## Which to choose

- **`init`** ‚Äî first paint, declarative, just trackIds. Default.
- **Top-level `location`** ‚Äî when you only need a starting region and don't need
  to open any tracks declaratively.
- **`showTrack`** ‚Äî runtime, imperative, composes with React state.
- **Full `defaultSession` snapshot** ‚Äî only when round-tripping an exported
  session.
