# Initialization

The `init` field under `defaultSession.view` is a *simplified* way to
specify a default session. It lets you write a few short fields —
which assembly, where to land, which tracks to open — instead of the
verbose snapshot format that `defaultSession.view.tracks` requires.

## `init` vs full `defaultSession`

Both forms below open the same RefSeq genes track at the same locus on
GRCh38. Compare the verbosity:

### With `init` (simple)

```js
defaultSession: {
  name: 'My session',
  view: {
    type: 'LinearGenomeView',
    init: {
      assembly: 'hg38',
      loc: 'chr1:11,106,077-11,261,675',
      tracks: ['ncbi-refseq-genes'],
    },
  },
}
```

You name the trackIds you want open. JBrowse resolves them after the
assembly loads, picks sensible default displays, and applies the
location.

### Equivalent full snapshot

```js
defaultSession: {
  name: 'My session',
  view: {
    id: 'linearGenomeView',
    type: 'LinearGenomeView',
    bpPerPx: 188.34,
    offsetPx: 156721,
    displayedRegions: [
      {
        refName: 'chr1',
        start: 0,
        end: 248956422,
        assemblyName: 'hg38',
      },
    ],
    tracks: [
      {
        type: 'FeatureTrack',
        configuration: 'ncbi-refseq-genes',
        displays: [
          {
            type: 'LinearBasicDisplay',
            configuration: 'ncbi-refseq-genes-LinearBasicDisplay',
          },
        ],
      },
    ],
  },
}
```

You provide IDs at every level — track configuration, display type,
display configuration — and you compute (or guess at) `bpPerPx` and
`offsetPx` for the location instead of giving a locstring.

The verbose form exists for a reason: it's exactly what gets serialized
when you save or export a session, so it can round-trip arbitrary view
state. Reach for it only when you are loading a session you previously
exported (see [Default Sessions](?path=/docs/default-sessions--docs)).
For everything else, use `init`.

## The `init` shape

```js
init: {
  assembly: 'hg38',                 // assembly name or alias
  loc: 'chr1:11,106,077-11,261,675', // any locstring
  tracks: ['my-gene-track-id'],      // trackIds to open
}
```

Sibling fields on `defaultSession.view` configure the view itself —
`showCytobandsSetting`, `showGridlines`, `colorByCDS`, etc.

## Loading alignments via `showTrack`

When tracks need to open in response to runtime conditions (a button
click, a search hit, a parent component prop) rather than at first
mount, prefer the imperative `viewState.session.view.showTrack(trackId)`
method. The example below adds a CRAM exome alignments track on top of
GRCh38, opened via `showTrack` after `createViewState` returns:

<Story of={HumanExomeExample} />

CRAM tracks (and BAM files without MD tags) require a `sequenceAdapter`
block on the track config so the worker can decode reads against the
reference — see the source for the exact shape.

## Which to choose

- **`init`** — first paint, declarative, just trackIds. Default.
- **`showTrack`** — runtime, imperative, composes with React state.
- **Full `defaultSession` snapshot** — only when round-tripping an
  exported session.
