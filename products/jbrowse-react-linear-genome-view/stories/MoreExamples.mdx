# More examples

This page collects additional examples that show the variety of things possible
with the LGV component.

## Real data: human exome

A live view of the GRCh38 assembly with NCBI RefSeq gene annotations and an
NA12878 exome CRAM track. Note that `CramAdapter` requires a `sequenceAdapter`
field so it can decode the reference-compressed reads.

You can render multiple independent `JBrowseLinearGenomeView` instances on the

## Programmatic navigation

### Navigate via locstring

Call `view.navToLocString(locstring, assemblyName)` to navigate from outside the
component. The call is async and can throw, so always catch errors.

<Story of={ExternalNavigateLocstring} />

### Navigate via region object

`view.navToLocations([{ refName, start, end, assemblyName }])` accepts an array
of region objects — useful when you have structured coordinates rather than a
locstring.

<Story of={ExternalNavigateObject} />

## Horizontally flipping the view

There are three ways to render a region in the reverse-complement orientation.

### Via locstring `[rev]` suffix

Append `[rev]` to a locstring passed to the `location` option or
`navToLocString`.

<Story of={HorizontallyFlippedViaLocstring} />

### Via an external button

Call `state.session.view.horizontallyFlip()` at any time to toggle the
orientation. This is useful when you want a flip button outside the JBrowse UI.

<Story of={HorizontallyFlippedViaButton} />

### Via `displayedRegions` in `defaultSession`

Set `reversed: true` on a region in `displayedRegions` inside your
`defaultSession` to start in flipped orientation.

<Story of={HorizontallyFlippedViaDisplayedRegions} />

### Multi-region intron-collapsed view with flip

This example computes an intron-collapsed (exon-only) view of a transcript by
building a list of padded exon regions from a feature object, joins them into a
multi-region locstring for `init.loc`, and adds a flip button. This pattern is
the programmatic equivalent of the built-in "Collapse introns" right-click action.

<Story of={WithMultipleDisplayedRegionsFlipped} />

Use a plugin and `Core-extendPluggableElement` to override the `zoomTo` and
`scrollTo` actions on `LinearGenomeView` with no-ops, effectively locking the
view to a fixed region.
