# Default sessions

When using JBrowse 2, information about the current app state such as what views
are open is stored in a session. It is possible to configure a JB2 app with a
`defaultSession`, which will be loaded when the app first loads.

## The `init` shorthand (recommended)

For most cases the `init` field is the simplest way to set an initial location
and open tracks. It is covered in detail in the
[Getting Started](?path=/docs/getting-started--docs) guide. Use the full session
format below only when you need precise control over display-level options.

## Creating a full default session

You can programmatically configure a `defaultSession`, but note that the session
format is an internal representation and may be verbose.

Here is the structure of the session:

```js
const defaultSession = {
  name: 'My session',
  view: {
    id: 'linearGenomeView',
    type: 'LinearGenomeView',
    tracks: [
      {
        // the first track displayed
      },
      {
        // the second track displayed
      },
      {
        // ...
      },
    ],
  },
}
```

The general outline for a track looks like this:

```js
{
  type: '{the type of the track}',
  configuration: '{the id of the track}',
  displays: [
    {
      type: '{the type of the display}',
      configuration:
        '{id of the track}-{the type of the display}',
    },
  ],
},
```

For example, to display a ReferenceSequenceTrack with the id
`GRCh38-ReferenceSequenceTrack`, the track configuration would be:

```js
{
  type: 'ReferenceSequenceTrack',
  configuration: 'GRCh38-ReferenceSequenceTrack',
  displays: [
    {
      type: 'LinearReferenceSequenceDisplay',
      configuration:
        'GRCh38-ReferenceSequenceTrack-LinearReferenceSequenceDisplay',
    },
  ],
}
```

## Exporting sessions from JBrowse Web/Desktop for use in @jbrowse/react-linear-genome-view2

A useful way to build default sessions is to configure the view interactively in
JBrowse 2 and then export it (File → Export session). This exports a JSON file
of the current session state, which you can adapt for the LGV component.

Load the tracks you want, open a single linear genome view, export the session,
and save it as `session.json` in your app.

To use the view settings in your app, you can create the `defaultSession` like
this:

```js
import session from './session.json'

const defaultSession = {
  name: 'My Session',
  view: session.session.views[0],
}
```
