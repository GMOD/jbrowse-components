# Multiple linear genome views

Two or more `JBrowseLinearGenomeView` instances can coexist on a single
page — each gets its own `viewState` from `createViewState`, and they
share no navigation, tracks, or session state. Useful for side-by-side
locus comparisons, multi-assembly dashboards, or report-style layouts
with several small embedded views.

<Story of={WithTwoLinearGenomeViews} />

If you need the two views to track each other (linked panning, shared
zoom), wire it up yourself via the MobX state tree — wrap your sibling
component in `observer`, read one view's `bpPerPx`/`offsetPx`, and call
the matching actions on the other. Both `viewState` objects are
ordinary state-tree nodes, so anything one view exposes can be mirrored
onto another.
