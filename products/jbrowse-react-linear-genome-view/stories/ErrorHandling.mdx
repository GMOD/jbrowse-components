# Error handling

The embedded views have no error boundary by default, so you must add your own.
Because `createViewState` can throw complex mobx-state-tree errors, we recommend
wrapping it with
[`@jbrowse/core/ui/ErrorMessage`](https://github.com/GMOD/jbrowse-components/blob/main/packages/core/ui/ErrorMessage.tsx),
which has specialized rendering for JBrowse errors (it is not a React error
boundary itself).

The example below shows a red error box for the BadTrack track type failing
validation.
