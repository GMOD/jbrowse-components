# Web worker RPC

By default the embedded view runs all data parsing and rendering on the main
thread, which can cause noticeable hitches on large alignments datasets (BAM,
CRAM). Switching to the WebWorker RPC moves that work off the main thread and
dramatically reduces UI stalls.

To enable it, import `makeWorkerInstance` and pass it to `createViewState`.
JBrowse handles the rest — feature parsing, layout, and rendering all run in the
worker.

A working integration with create-react-app is at
[jbrowse.org/jb2/docs/embedded_components](https://jbrowse.org/jb2/docs/embedded_components/).

## Bundler notes

**Webpack** — set `output.publicPath: 'auto'` so workers resolve their own URL
correctly. See
[this comment](https://github.com/webpack/webpack/issues/13791#issuecomment-897579223)
and the [official guide](https://webpack.js.org/guides/web-workers/).

**Other bundlers** — the worker is constructed via
`new Worker(new URL(...), { type: 'module' })`, which Vite handles natively.
Older bundlers may not.

The web worker is not enabled by default precisely because of these bundler
requirements, but we recommend turning it on whenever your toolchain supports
it.

## Using plugins with workers

When you load JBrowse plugins, they must be registered on **both** the main
thread and the worker. See [Using Plugins](?path=/docs/using-plugins--docs) and
[this PR](https://github.com/GMOD/jbrowse-react-linear-genome-view-cra5-demo/pull/1/files)
for the full pattern.
