# Observing view state

The view model is a [MobX-state-tree](https://mobx-state-tree.js.org) node, so
anything outside the LGV can subscribe to its state with `mobx-react`'s
`observer` HOC and re-render when relevant fields change. This is how you build
companion panels — coordinate readouts, feature inspectors, summary tables —
that stay in sync with the view without manual event wiring.

## Visible regions

`view.dynamicBlocks` describes the regions currently scrolled into view, updated
on every pan and zoom. An observer reading it gets free reactivity:

## Visible features

Reading actual feature data requires a round-trip through the RPC manager (the
same path the renderer uses). The example queries features for the visible
region inside an effect that reruns when the region changes.

<Story of={WithObserveVisibleFeatures} />

For frequently-changing state, debounce or throttle your queries —
`dynamicBlocks` updates every animation frame during a drag, and you likely
don't want a fetch per frame.
