# Styling and embedding

This page covers how the view looks and how it integrates into a host page:
theme colors, track display settings, composing with surrounding CSS, locking
down interaction, the side drawer, and Shadow DOM isolation.

## Track display settings

Per-track appearance — color, height, display mode, jexl color callbacks — lives
on a track's **displays**. For the common case, put your settings in a
`displays` object and JBrowse routes each one to the display that uses it. You
don't have to name the display type or write the `displays` array:

```js
{
  type: 'FeatureTrack',
  trackId: 'volvox_genes_green',
  name: 'Volvox genes',
  assemblyNames: ['volvox'],
  adapter: {
    /* ... */
  },
  // routes by slot name — `color` lands on whichever display defines it
  displays: { color: 'green' },
}
```

A `jexl:` expression works the same way for per-feature coloring, e.g.
`displays: { color: "jexl:get(feature,'type')=='CDS'?'red':'blue'" }`. For full
control — different settings for two displays that share a slot, an explicit
`displayId`, choosing the default display — pass `displays` as the array form
instead (see the
[track config guide](https://jbrowse.org/jb2/docs/config_guides/tracks/)).

## Theming

The embedded view accepts a Material-UI-style theme on its `configuration`
block. JBrowse defines four named palette colors — `primary`, `secondary`,
`tertiary`, and `quaternary` — that drive most of the chrome (toolbars, buttons,
highlights). The DNA-base colors used in the sequence display are configurable
separately. See the
[theming guide](https://jbrowse.org/jb2/docs/config_guides/theme/) for the full
set of options.

## Composing with parent styles

The embedded view inherits CSS from its host. There's no shadow-DOM isolation in
the default rendering path (see [Shadow DOM](#shadow-dom) below if you need
that), so wrapping the component in a styled container composes normally —

## Customizing interaction

The embedded view is interactive by default — wheel zoom, click-drag pan, the
full track-selector UI. The settings below cover the most common adjustments
when embedding into a host page that wants tighter control.

### Locking down the track list

By default users can add tracks via the UI. Pass `disableAddTracks: true` to
hide the "+ Add track" button — this also disables on-the-fly track creation
features like sequence-search tracks and multi-wiggle tracks.

```js
const state = createViewState({
  assembly,
  tracks,
  location: 'ctgA:1,000..5,000',
  disableAddTracks: true,
})
```

## Drawer widget

By default, widgets like the hierarchical track selector open in a modal dialog.
Passing `drawerViewHeight` to `createViewState` switches widgets to render in a
resizable side drawer alongside the view, which matches the behavior of the full
JBrowse Web app.

```js
const state = createViewState({
  assembly,
  tracks,
  location: 'ctgA:1105..1221',
  drawerViewHeight: '100vh',
  defaultSession: {
    /* ... */
  },
})
```

`drawerViewHeight` accepts any CSS height value (`'100vh'`, `'600px'`, `'80%'`,
etc.). It constrains the height of the embedded view's grid container while a
drawer is open, giving the drawer a fixed scroll region. When no drawer widget
is visible, the view sizes to its parent as usual — the height override applies
only when needed.

The drawer is resizable (drag the handle), repositionable left ↔ right via the ⋮
menu, minimizable, and closeable. With no drawer widget visible, widget popups
fall back to the modal presentation.

## Shadow DOM

`@jbrowse/react-linear-genome-view2` can be mounted inside a Shadow DOM — useful
when embedding into a host page that has its own conflicting global styles, or
when packaging the view as a custom element.

The component itself doesn't require any special configuration; the example
below shows it rendered inside a shadow root via React's portal mechanism.
Styles created inside the shadow root (Material-UI's emotion output) stay scoped
to the root, and styles outside it cannot leak in.
