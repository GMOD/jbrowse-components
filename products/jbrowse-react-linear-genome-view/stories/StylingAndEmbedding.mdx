# Styling and embedding

This page covers how the view looks and how it integrates into a host page:
theme colors, track display settings, composing with surrounding CSS, locking
down interaction, the side drawer, and Shadow DOM isolation.

## Track display settings

Per-track appearance â€” color, height, display mode, jexl color callbacks â€” is
set on a track's **displays** (the different ways a track can be drawn). The
quick way: rather than writing out the full `displays` array, list your settings
in a `displays` object. JBrowse works out which display each setting belongs to
and applies it for you, so you don't have to know the display's name:

```js
{
  type: 'FeatureTrack',
  trackId: 'volvox_genes_green',
  name: 'Volvox genes',
  assemblyNames: ['volvox'],
  adapter: {
    /* ... */
  },
  // JBrowse applies this to the track's LinearBasicDisplay for you
  displays: { color: 'green' },
}
```

A `jexl:` expression works the same way for per-feature coloring, e.g.
`displays: { color: "jexl:get(feature,'type')=='CDS'?'red':'blue'" }`. For full
control â€” giving two displays different values for the same setting, an explicit
`displayId`, or choosing the default display â€” pass `displays` as the array form
instead (see the
[track config guide](https://jbrowse.org/jb2/docs/config_guides/tracks/)).

### Per-feature colors and labels with jexl

The same `jexl:` mechanism drives both feature **color** and feature **labels**.
A jexl expression has the `feature` in scope and is evaluated per feature, so
you can color and re-label features from their own attributes with no extra UI
or plugin code â€” `color` reads a CSS color, `labels.name` reads the displayed
text:

```js
displays: {
  // color by strand
  color: "jexl:get(feature,'strand')==1?'#1f77b4':'#d62728'",
  // label with the feature's name and type
  labels: { name: "jexl:get(feature,'name')+' ['+get(feature,'type')+']'" },
}
```

## Theming

The embedded view accepts a Material-UI-style theme on its `configuration`
block. JBrowse defines four named palette colors â€” `primary`, `secondary`,
`tertiary`, and `quaternary` â€” that drive most of the chrome (toolbars, buttons,
highlights). The DNA-base colors used in the sequence display are configurable
separately. See the
[theming guide](https://jbrowse.org/jb2/docs/config_guides/theme/) for the full
set of options.

## Customizing interaction

The embedded view is interactive by default â€” wheel zoom, click-drag pan, the
full track-selector UI. The settings below cover the most common adjustments
when embedding into a host page that wants tighter control.

### Locking down the track list

By default users can add tracks via the UI. Pass `disableAddTracks` to hide the
"+ Add track" button â€” this also disables on-the-fly track creation features
like sequence-search tracks and multi-wiggle tracks.

```jsx
<LinearGenomeView
  assembly={assembly}
  tracks={tracks}
  init={{ loc: 'ctgA:1,000..5,000' }}
  disableAddTracks
/>
```

## Drawer widget

By default, widgets like the hierarchical track selector open in a modal dialog.
Passing `drawerViewHeight` to `createViewState` switches widgets to render in a
resizable side drawer alongside the view, which matches the behavior of the full
JBrowse Web app.

```js
const state = createViewState({
  assembly,
  tracks,
  location: 'ctgA:1105..1221',
  drawerViewHeight: '100vh',
  defaultSession: {
    /* ... */
  },
})
```

`drawerViewHeight` accepts any CSS height value (`'100vh'`, `'600px'`, `'80%'`,
etc.). It constrains the height of the embedded view's grid container while a
drawer is open, giving the drawer a fixed scroll region. When no drawer widget
is visible, the view sizes to its parent as usual â€” the height override applies
only when needed.

The drawer is resizable (drag the handle), repositionable left â†” right via the â‹®
menu, minimizable, and closeable. With no drawer widget visible, widget popups
fall back to the modal presentation.

when embedding into a host page that has its own conflicting global styles, or
