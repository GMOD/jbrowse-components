## An overview of the LGV API

The LGV component relies on the same
[configuration](https://jbrowse.org/jb2/docs/config_guide) as the main JB2 app,
with a few noteworthy differences.

First, the component can be configured using a standalone JSON file like JB2, or
can be configured with JavaScript objects in the source code of the app where it
is being used, as shown
[here](https://github.com/GMOD/jbrowse-components/blob/HEAD/products/jbrowse-react-linear-genome-view/docs/example.md).

Second, while this is the structure of a JB2 configuration file:

```json
{
  "configuration": {
    /* global configs here */
  },
  "assemblies": [
    /* list of assembly configurations, e.g. the genomes being viewed */
  ],
  "tracks": [
    /* array of tracks being loaded, contain reference to which assembl(y/ies)
    they belong to */
  ],
  "defaultSession": {
    /* optional default session */
  }
}
```

The LGV component only accepts a single assembly object instead of an array:

```json
{
  "assembly": {
    /* the assembly being used for the browser */
  }
}
```

You can also set an initial location when the component first loads, by passing
it to `createViewState`:

```js
const state = createViewState({
  assembly,
  tracks,
  location: '10:29,838,737..29,838,819',
  defaultSession,
})
```

## Disabling the ability to add tracks

By default users can add tracks, but the `disableAddTracks` flag prevents this.
Note that it also prevents creating sequence search tracks or multi-wiggle tracks.

```js
const state = createViewState({
  assembly,
  tracks,
  location: '10:29,838,737..29,838,819',
  defaultSession,
  disableAddTracks: true,
})
```

## Opening tracks with the `init` shorthand

The recommended way to set an initial location and open tracks is the `init`
field inside `defaultSession.view`. It avoids constructing a full
`displayedRegions` array and is much more concise than a complete session object:

```js
const state = createViewState({
  assembly,
  tracks,
  defaultSession: {
    name: 'My session',
    view: {
      type: 'LinearGenomeView',
      init: {
        loc: 'chr1:11,106,077-11,261,675',
        assembly: 'hg38',
        tracks: ['trackId1', 'trackId2'],
      },
    },
  },
})
```

`init.loc` accepts any locstring (including multi-region, e.g.
`'chr1:100-200 chr1:500-600'`). `init.tracks` is a list of trackIds to open.

## Using the useCreateViewState hook

`createViewState` is an expensive operation and should not be called on every
render. If your component re-renders (e.g. because a parent component has
state), calling `createViewState` directly in the component body will recreate
the entire browser state on each render.

The `useCreateViewState` hook wraps `createViewState` so the state is created
only once per component mount, regardless of how many times the component
re-renders:

```js
import { useCreateViewState, JBrowseLinearGenomeView } from '@jbrowse/react-linear-genome-view2'

function GenomeView() {
  const state = useCreateViewState({ assembly, tracks, location: '...' })
  return <JBrowseLinearGenomeView viewState={state} />
}
```

The example below has a counter in the parent. Clicking "Re-render parent"
increments the counter and triggers a re-render, but the genome view is
unaffected because `useCreateViewState` keeps the state stable.
